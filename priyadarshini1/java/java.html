Package

Namespace for organizing related classes

A package in Java is a namespace that organizes a set of related classes and interfaces. Conceptually, you can think of packages as being similar to different folders on your computer.

Usage:

  • Preventing Naming Conflicts: Packages help avoid class name conflicts. For example, two classes named Employee can be placed in different packages.
  • Controlled Access: Packages can help define access controls. Classes, fields, and methods declared with no access modifier are accessible only within the same package.
  • Encapsulation: Packages can hide classes, methods, and fields by not exposing them to other packages.
  • Easy Maintenance: Grouping related classes and interfaces in packages makes it easier to locate and maintain the code.

Class

Blueprint for creating objects

A class in Java is a blueprint from which individual objects are created. A class can contain:

  • Fields: Variables that hold the state of the objects.
  • Methods: Functions that define the behavior of the objects.
  • Constructors: Special methods that initialize new objects.
  • Blocks: Code blocks that are executed during the object creation.
  • Nested Class and Interface: Classes and interfaces defined within another class.
Class Example 
public class Employee {
    // Fields (instance variables)
    private String name;
    private int id;
    private double salary;
    
    // Constructor
    public Employee(String name, int id, double salary) {
        this.name = name;
        this.id = id;
        this.salary = salary;
    }
    
    // Method
    public void displayDetails() {
        System.out.println("Employee: " + name + ", ID: " + id);
    }
}

Main Method

Entry point of a Java application

The main() method is the entry point of a Java application. It is always written as:

Main Method Signature 
public static void main(String[] args) {
    // Application code goes here
}

Components:

  • public: The method is accessible from anywhere.
  • static: The method can be called without creating an instance of the class.
  • void: The method does not return any value.
  • String[] args: The parameter is an array of String arguments passed to the main method.

Polymorphism

One interface, multiple implementations

Polymorphism in Java allows methods to do different things based on the object it is acting upon.

Types of Polymorphism:

  • Compile-Time Polymorphism: Achieved using method overloading.
  • Runtime Polymorphism: Achieved using method overriding.
Polymorphism Example 
// Method Overloading (Compile-Time Polymorphism)
class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    
    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

// Method Overriding (Runtime Polymorphism)
class Animal {
    public void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("Dog barks");
    }
}

JDK, JRE, JVM

Java Platform Components

JDK (Java Development Kit)

JDK is a software development kit used to develop Java applications and applets. It includes the JRE, an interpreter/loader (Java), a compiler (javac), an archiver (jar), a documentation generator (Javadoc), and other tools needed for Java development.

JRE (Java Runtime Environment)

JRE provides the libraries, Java Virtual Machine (JVM), and other components necessary to run applications written in Java. It does not include development tools like compilers or debuggers.

JVM (Java Virtual Machine)

JVM is an abstract machine that enables your computer to run a Java program. It provides a runtime environment in which Java bytecode can be executed.

Relationship Between JDK, JRE, and JVM

  • JDK contains the JRE, which in turn contains the JVM.
  • The JDK is used for development, containing the tools necessary for developing Java programs.
  • The JRE is used to run Java applications.
  • The JVM is the engine that runs the bytecode generated by the Java compiler.

Collections API

Framework for storing and manipulating groups of data

Provides a set of classes and interfaces for storing and manipulating groups of data as a single unit.

Main Interfaces:

  • List: Ordered collection (e.g., ArrayList, LinkedList).
  • Set: Unordered collection with no duplicates (e.g., HashSet, TreeSet).
  • Queue: Collection used to hold elements prior to processing (e.g., LinkedList, PriorityQueue).
  • Map: Key-value pairs (e.g., HashMap, TreeMap).
Collections Example 
// ArrayList Example
List<String> names = new ArrayList<>();
names.add("John");
names.add("Jane");
names.add("Alice");

// HashSet Example
Set<Integer> numbers = new HashSet<>();
numbers.add(10);
numbers.add(20);
numbers.add(10); // Duplicate, won't be added

// HashMap Example
Map<String, Integer> ages = new HashMap<>();
ages.put("John", 25);
ages.put("Jane", 30);

Stream API (Java 8+)

Introduced in Java 8, the Stream API is used to process collections of objects in a functional style.

Stream API Example 
List<String> names = Arrays.asList("John", "Jane", "Adam", "Eve");

// Filter names starting with 'J' and convert to uppercase
List<String> result = names.stream()
    .filter(name -> name.startsWith("J"))
    .map(String::toUpperCase)
    .collect(Collectors.toList());

// result = ["JOHN", "JANE"]
